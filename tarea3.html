01 — Introducción

¿Qué es HTML5?

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. La versión 5, lanzada oficialmente en 2014, incorporó soporte nativo para multimedia, nuevas etiquetas semánticas y APIs modernas.

🌐
Lenguaje de marcado
No es un lenguaje de programación. Define la estructura y el significado del contenido mediante etiquetas.
📄
Archivo .html
Los documentos HTML se guardan con extensión .html y son interpretados por los navegadores web.
🏗️
Estructura base
Todo documento HTML tiene una jerarquía obligatoria: DOCTYPE, html, head y body.
🔖
Etiquetas
Las etiquetas son palabras clave entre corchetes angulares que le indican al navegador cómo mostrar el contenido.
🎨
Trabaja con CSS y JS
HTML define la estructura, CSS el estilo visual y JavaScript la interactividad. Los tres forman la tríada del desarrollo web.
♿
Accesibilidad
Un HTML bien escrito es accesible para lectores de pantalla y tecnologías de asistencia.
Tip: Para comenzar solo necesitas un editor de texto (como VSCode o Notepad++) y un navegador web. No hace falta instalar nada más.

02 — Fundamentos

Estructura base de un documento

Todo documento HTML5 válido debe comenzar con una declaración DOCTYPE y contener las etiquetas raíz obligatorias.

<!-- Declaración del tipo de documento -->
<!DOCTYPE html>

<html lang="es">
<!-- Metadatos: no visibles en pantalla -->
<head>
<meta charset="UTF-8"/>
<meta name="viewport" content="width=device-width"/>
<title>Título de la página</title>
</head>

<!-- Contenido visible por el usuario -->
<body>
Todo el contenido visible va aquí...
</body>
</html>
<!DOCTYPE html>Declaración
Indica al navegador que el documento usa HTML5. Siempre va en la primera línea. No es una etiqueta HTML sino una instrucción.
<html>Raíz
Elemento raíz que engloba todo el documento. El atributo lang define el idioma del contenido.
<head>Metadatos
Contiene información sobre el documento que no se muestra al usuario: título, codificación, enlaces a CSS, etc.
<body>Contenido
Todo el contenido visible de la página va dentro de esta etiqueta: textos, imágenes, botones, formularios, etc.
<meta>Metadato
Define metadatos del documento. charset="UTF-8" asegura que los caracteres especiales (acentos, ñ) se muestren correctamente.
<title>Metadato
Define el título que aparece en la pestaña del navegador y en los resultados de búsqueda (SEO).

03 — Tipografía

Etiquetas de texto y tipografía

Las etiquetas de texto definen la jerarquía, énfasis y presentación del contenido escrito de la página.

<h1> – <h6>Encabezados
Seis niveles de encabezado. H1 es el más importante (título principal) y H6 el menor. Solo debe haber un H1 por página.
<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>
<p>Bloque
Define un párrafo de texto. El navegador añade espacio automáticamente antes y después.
<p>Este es un párrafo de texto con información relevante.</p>
<strong>Inline
Texto de importancia alta, se muestra en negrita. Tiene valor semántico (importante para SEO y accesibilidad).
<p>Esto es <strong>muy importante</strong>.</p>
<em>Inline
Énfasis en el texto, se muestra en cursiva. Semánticamente indica énfasis, no solo estilo visual.
<p>El resultado <em>podría</em> cambiar.</p>
<span>Inline
Contenedor en línea sin valor semántico. Útil para aplicar estilos CSS a partes de un texto.
<p>Color <span style="color:red">rojo</span></p>
<br>Inline
Inserta un salto de línea. Es una etiqueta vacía (no tiene cierre). Úsala solo cuando el salto sea parte del contenido.
<p>Línea uno<br/>Línea dos</p>
<blockquote>Bloque
Indica una cita larga de otra fuente. Se muestra con sangría. Usa el atributo cite para indicar la fuente.
<blockquote cite="url">
Texto citado aquí.
</blockquote>
<code>Inline
Muestra fragmentos de código con fuente monoespaciada. Para bloques de código largos se combina con <pre>.
<p>Usa <code>console.log()</code></p>
textos.html
<h1>Cultura Digital</h1>
<h2>Módulo 1: Fundamentos</h2>

<p>HTML es el <strong>lenguaje base</strong> de la web.
Fue creado por <em>Tim Berners-Lee</em> en 1991.</p>

<blockquote>
  La web es para todos. — Tim Berners-Lee
</blockquote>

<p>Para imprimir en consola usa <code>console.log()</code>.</p>

04 — Organización

Listas

Las listas son fundamentales para organizar información. HTML tiene tres tipos: listas no ordenadas, ordenadas y de definición.

<ul>Lista
Lista no ordenada (con viñetas). Úsala cuando el orden de los elementos no importa.
<ul>
<li>HTML</li>
<li>CSS</li>
<li>JavaScript</li>
</ul>
<ol>Lista
Lista ordenada (numerada). Úsala cuando el orden sí importa, como pasos de un proceso.
<ol>
<li>Abrir editor</li>
<li>Escribir código</li>
<li>Guardar archivo</li>
</ol>
<li>Item
Define cada elemento de una lista. Siempre debe estar dentro de <ul>, <ol> o <menu>.
<li>Elemento de lista</li>
<dl> <dt> <dd>Definición
Lista de definición. <dt> es el término y <dd> su definición. Ideal para glosarios.
<dl>
<dt>HTML</dt>
<dd>Lenguaje de marcado</dd>
</dl>

05 — Navegación y media

Enlaces e imágenes

Los hipervínculos son la esencia de la web. Las imágenes enriquecen visualmente el contenido. Ambos requieren atributos específicos.

<a>Enlace
Crea un hipervínculo. El atributo href define el destino. Con target="_blank" abre en nueva pestaña.
<a href="https://ejemplo.com"
target="_blank">
Visitar sitio
</a>
<img>Imagen
Inserta una imagen. Es una etiqueta vacía (sin cierre). src es la ruta y alt el texto alternativo (obligatorio).
<img src="foto.jpg"
alt="Descripción"
width="300"/>
<figure>Semántico
Envuelve contenido visual (imágenes, diagramas) junto con su descripción usando <figcaption>.
<figure>
<img src="grafica.png" alt="Gráfica"/>
<figcaption>Fig. 1</figcaption>
</figure>
<link>Head
Va dentro de <head>. Vincula recursos externos como hojas de estilo CSS o iconos de la página.
<link rel="stylesheet"
href="estilos.css"/>
Importante: Siempre agrega el atributo alt a las imágenes. Es obligatorio para accesibilidad y mejora el SEO de la página.

06 — HTML5 Semántico

Etiquetas semánticas de HTML5

HTML5 introdujo etiquetas que describen el significado del contenido, no solo su apariencia. Esto mejora la accesibilidad, el SEO y la legibilidad del código.

¿Qué es la semántica?
Antes de HTML5 se usaban <div> para todo. Las etiquetas semánticas reemplazan esos divs genéricos con etiquetas que describen qué tipo de contenido contienen, lo que ayuda a los navegadores, lectores de pantalla y motores de búsqueda a entender la estructura de la página.
<header>Semántico
Encabezado de la página o de una sección. Generalmente contiene el logo, navegación principal y el título del sitio.
<nav>Semántico
Bloque de navegación principal. Contiene los enlaces de menú del sitio. Solo para navegación principal, no para todos los enlaces.
<main>Semántico
Contenido principal de la página. Solo debe haber uno por documento. No incluir header, footer ni nav aquí.
<section>Semántico
Agrupa contenido temáticamente relacionado. Debe tener su propio encabezado. Alternativa semántica al <div>.
<article>Semántico
Contenido independiente y reutilizable: una entrada de blog, una noticia, un comentario. Puede existir fuera de contexto.
<aside>Semántico
Contenido relacionado pero secundario al principal: barras laterales, publicidad, links relacionados.
<footer>Semántico
Pie de página de la página o de una sección. Contiene información de copyright, contacto y enlaces secundarios.
<div>Genérico
Contenedor genérico sin valor semántico. Úsalo solo cuando ninguna etiqueta semántica aplique. Es para agrupar con CSS.
semantica.html
<body>
  <header>
    <h1>Mi Portafolio</h1>
    <nav>
      <a href="#inicio">Inicio</a>
      <a href="#tareas">Tareas</a>
    </nav>
  </header>

  <main>
    <section id="tareas">
      <h2>Mis Tareas</h2>
      <article>
        <h3>Tarea 1</h3>
        <p>Descripción...</p>
      </article>
    </section>
  </main>

  <footer>
    <p>© 2025 Mi Portafolio</p>
  </footer>
</body>

07 — Interacción

Formularios

Los formularios permiten recoger información del usuario. HTML5 introdujo nuevos tipos de campos que incluyen validación automática en el navegador.

<form>Contenedor
Elemento raíz de un formulario. El atributo action define a dónde enviar los datos y method cómo enviarlos (GET o POST).
<form action="/enviar" method="POST">
...
</form>
<input>Campo
Campo de entrada de datos. El atributo type define su comportamiento: text, email, password, number, date, checkbox, radio, file...
<input type="email"
placeholder="tu@correo.com"
required/>
<label>Accesibilidad
Etiqueta asociada a un campo. Mejora la accesibilidad. Con for vincula al id del input.
<label for="nombre">Nombre:</label>
<input id="nombre" type="text"/>
<textarea>Campo
Área de texto multilínea. Los atributos rows y cols definen su tamaño inicial.
<textarea rows="4"
placeholder="Tu mensaje...">
</textarea>
<select>Campo
Menú desplegable. Las opciones se definen con <option>. El atributo multiple permite selección múltiple.
<select name="lenguaje">
<option value="html">HTML</option>
<option value="css">CSS</option>
</select>
<button>Acción
Botón interactivo. type="submit" envía el formulario. type="button" lo deja para JavaScript.
<button type="submit">
Enviar formulario
</button>

08 — Multimedia

Multimedia en HTML5

HTML5 incorporó soporte nativo para audio y video sin necesidad de plugins externos como Flash.

<video>Media
Inserta un reproductor de video. Atributos clave: controls, autoplay, loop, muted, poster.
<video controls width="400">
<source src="video.mp4" type="video/mp4"/>
Tu navegador no soporta video.
</video>
<audio>Media
Inserta un reproductor de audio. Soporta múltiples formatos: MP3, OGG, WAV. Usa <source> para compatibilidad.
<audio controls>
<source src="sonido.mp3" type="audio/mp3"/>
</audio>
<iframe>Embed
Incrusta otro documento HTML o contenido externo (YouTube, Google Maps) dentro de la página.
<iframe
src="https://youtube.com/embed/id"
width="560" height="315"
allowfullscreen>
</iframe>
<canvas>Gráficos
Área de dibujo programable mediante JavaScript. Permite crear gráficos, animaciones y juegos 2D directamente en el navegador.
<canvas id="miCanvas"
width="400" height="300">
</canvas>

09 — Datos

Tablas

Las tablas se usan para mostrar datos relacionales en filas y columnas. No deben usarse para maquetar páginas, eso es tarea del CSS.

tabla.html
<table>
  <!-- Encabezado de la tabla -->
  <thead>
    <tr>
      <th>Lenguaje</th>
      <th>Tipo</th>
      <th>Año</th>
    </tr>
  </thead>

  <!-- Cuerpo de la tabla -->
  <tbody>
    <tr>
      <td>HTML</td>
      <td>Marcado</td>
      <td>1991</td>
    </tr>
    <tr>
      <td>CSS</td>
      <td>Estilos</td>
      <td>1996</td>
    </tr>
  </tbody>

  <!-- Pie de tabla (opcional) -->
  <tfoot>
    <tr>
      <td colspan="3">Fuente: MDN Web Docs</td>
    </tr>
  </tfoot>
</table>
EtiquetaFunción
<table>Contenedor principal de la tabla
<thead>Agrupa las filas del encabezado
<tbody>Agrupa las filas del cuerpo de datos
<tfoot>Agrupa las filas del pie de tabla
<tr>Define una fila (table row)
<th>Celda de encabezado (negrita, centrado por defecto)
<td>Celda de dato normal
colspan / rowspanFusionar columnas o filas

10 — Atributos

Atributos globales

Los atributos globales pueden aplicarse a cualquier elemento HTML. Son la forma de pasar información adicional a las etiquetas.

AtributoDescripciónEjemplo
idIdentificador único en el documento. Usado por CSS y JavaScript.id="encabezado"
classClase CSS. Se pueden asignar múltiples clases separadas por espacio.class="btn btn-verde"
styleEstilos CSS en línea. Evitar cuando sea posible, usar CSS externo.style="color:red"
titleTexto descriptivo. Aparece como tooltip al pasar el cursor.title="Más información"
langIdioma del contenido del elemento.lang="es"
hiddenOculta el elemento visualmente y del árbol de accesibilidad.hidden
tabindexControla el orden de navegación con la tecla Tab.tabindex="1"
data-*Atributos personalizados para almacenar datos en el HTML.data-id="42"
contenteditablePermite al usuario editar el contenido del elemento.contenteditable="true"
draggableDefine si el elemento puede ser arrastrado con drag & drop.draggable="true"

11 — Calidad de código

Buenas prácticas

Escribir HTML de calidad va más allá de que funcione. Un buen código es legible, accesible, semántico y mantenible.

✅
Usa etiquetas semánticas
Prefiere <header>, <nav>, <main>, <article> sobre divs sin significado.
✅
Indenta tu código
Usa 2 o 4 espacios de sangría para cada nivel de anidamiento. Facilita la lectura y el mantenimiento.
✅
Alt en imágenes
Siempre incluye el atributo alt en <img>. Si la imagen es decorativa, usa alt="" vacío.
✅
Un solo H1 por página
El H1 es el título principal. Debe haber exactamente uno por página para buen SEO.
✅
Cierra tus etiquetas
Aunque algunos navegadores toleran etiquetas sin cerrar, siempre ciérralas para evitar comportamiento inesperado.
✅
Minúsculas siempre
Escribe los nombres de etiquetas y atributos en minúsculas. Es el estándar HTML5.
✅
Comillas en atributos
Siempre pon los valores de atributos entre comillas: href="url" no href=url.
✅
Valida tu HTML
Usa el validador oficial del W3C (validator.w3.org) para detectar errores en tu código.

12 — Práctica

Ejemplo completo

Aquí tienes un documento HTML5 completo que aplica todo lo aprendido: estructura, semántica, texto, listas, enlaces, imágenes y formulario.

portafolio.html
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>Mi Portafolio · Cultura Digital</title>
  <link rel="stylesheet" href="estilos.css"/>
</head>
<body>

  <header>
    <h1>Portafolio de Evidencias</h1>
    <nav>
      <a href="#sobre-mi">Sobre mí</a>
      <a href="#tareas">Tareas</a>
      <a href="#contacto">Contacto</a>
    </nav>
  </header>

  <main>
    <section id="sobre-mi">
      <h2>Sobre mí</h2>
      <figure>
        <img src="foto.jpg" alt="Mi foto de perfil" width="200"/>
        <figcaption>Estudiante de IA, 2° semestre</figcaption>
      </figure>
      <p>Soy estudiante de <strong>Inteligencia Artificial</strong>.</p>
    </section>

    <section id="tareas">
      <h2>Mis tareas</h2>
      <ul>
        <li><a href="tarea1.html">El padre del internet</a></li>
        <li><a href="tarea2.html">Lenguajes de programación</a></li>
        <li><a href="tarea3.html">Guía de HTML5</a></li>
      </ul>
    </section>

    <section id="contacto">
      <h2>Contacto</h2>
      <form action="/enviar" method="POST">
        <label for="nombre">Nombre:</label>
        <input id="nombre" type="text" required/>
        <label for="email">Email:</label>
        <input id="email" type="email" required/>
        <button type="submit">Enviar</button>
      </form>
    </section>
  </main>

  <footer>
    <p>© 2025 · Cultura Digital 2 · 2° Semestre de IA</p>
  </footer>

</body>
</html>
